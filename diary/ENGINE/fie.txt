# Fault Intelligence Engine (FIE)

**ID:** `fie`
**Name:** Fault Intelligence
**Description:** Diagnostic fault codes and maintenance

## Purpose
The FIE engine analyzes diagnostic fault codes, providing insights into vehicle health and maintenance needs. It stores and retrieves `fie_scores` in the database.

## Configuration Details

The FIE engine's operational parameters are defined as follows:

-   **Database Table:** `fie_scores`
-   **Score Field:** `score` (a numerical value between 0 and 100 representing fault severity or maintenance urgency)
-   **Required POST Fields:** `vehicle_id`, `client_id`, `score`
-   **Insertable Fields:**
    -   `vehicle_id`
    -   `client_id`
    -   `score`
    -   `active_fault_count`
    -   `critical_fault_count`
    -   `maintenance_due` (Boolean: 0 or 1)
    -   `computed_at` (Timestamp when the score was computed)
-   **Filterable GET Fields:**
    -   `client_id`
    -   `vehicle_id`

## Formulas and Logic

The FIE engine's logic involves:

-   **Score Validation:** The `score` field must be a number between 0 and 100.
-   **Timestamping:** If `computed_at` is not provided in a POST request, it defaults to the current UTC timestamp.
-   **Boolean Field Handling:** `maintenance_due` is stored as an INTEGER (0 for false, 1 for true) in the SQLite database.
-   **Data Filtering:** GET requests allow filtering by `client_id` and `vehicle_id`.
-   **Ordering and Limiting:** GET results are ordered by `computed_at` DESC and limited to a maximum of 500 records.
-   **Authorization:**
    -   GET requests require `admin` or `dev` roles.
    -   POST requests require the `dev` role.
-   **Audit Logging:** Every successful POST operation logs an audit event.
