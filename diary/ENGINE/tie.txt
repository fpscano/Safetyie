# Telematics Intelligence Engine (TIE)

**ID:** `tie`
**Name:** Telematics Intelligence
**Description:** Real-time location and speed telemetry

## Purpose
The TIE engine collects and analyzes real-time telematics data, including vehicle location and speed. It stores and retrieves `tie_scores` in the database, providing current and historical telemetry insights.

## Configuration Details

The TIE engine's operational parameters are defined as follows:

-   **Database Table:** `tie_scores`
-   **Score Field:** `score` (a numerical value between 0 and 100, possibly reflecting a telematics-derived metric like driving performance or efficiency)
-   **Required POST Fields:** `vehicle_id`, `client_id`, `score`
-   **Insertable Fields:**
    -   `vehicle_id`
    -   `client_id`
    -   `score`
    -   `avg_speed_mph`
    -   `max_speed_mph`
    -   `last_lat` (Last recorded latitude)
    -   `last_lng` (Last recorded longitude)
    -   `last_seen_at` (Timestamp of the last telemetry update)
    -   `computed_at` (Timestamp when the score was computed)
-   **Filterable GET Fields:**
    -   `client_id`
    -   `vehicle_id`

## Operational Mechanics (Data Flow)

The TIE engine processes data through GET and POST requests, similar to other engines.

### GET Request Flow

```
+----------------+       +-------------------+       +---------------------+
| Client Request | ----> | Engine API (/tie) | ----> | Authentication/Auth |
| (GET /tie)     |       | (handleGET)       |       | (getAuthUser)       |
+----------------+       +-------------------+       +---------------------+
        |                          |                          | (Success: admin/dev)
        |                          |                          v
        |                          |               +-----------------------+
        |                          |               | Validate client_id &  |
        |                          |               | other filter fields   |
        |                          |               +-----------------------+
        |                          |                          |
        |                          v                          v
        |                  +----------------------------------------------+
        |                  | Construct SQL Query (SELECT * FROM tie_scores)|
        |                  | WHERE client_id = ? AND vehicle_id = ? ...   |

        |                                        v

        |<------------------------- | Database (Cloudflare D1)|
        |    (jsonResponse)         | (env.DB.prepare.all)  |

        (JSON Array of TIE scores)
```

## Formulas and Logic

The TIE engine's logic involves:

-   **Score Validation:** The `score` field must be a number between 0 and 100.
-   **Timestamping:** If `computed_at` is not provided in a POST request, it defaults to the current UTC timestamp.
-   **Data Filtering:** GET requests allow filtering by `client_id` and `vehicle_id`.
-   **Ordering and Limiting:** GET results are ordered by `computed_at` DESC and limited to a maximum of 500 records.
-   **Authorization:**
    -   GET requests require `admin` or `dev` roles.
    -   POST requests require the `dev` role.
-   **Audit Logging:** Every successful POST operation logs an audit event.
